# Sprint 2 - Business Requirements and Current Environment Assessment

---

# Sprint Overview

Sprint 2 focuses on understanding the current Microsoft 365 environment before any Microsoft Intune implementation activities begin.

The sprint establishes a baseline of the existing environment, identifies business requirements, documents the current management approach, and highlights any gaps that must be addressed before implementation.

The outputs from this sprint will guide the architecture and technical implementation phases of the project.

---

# Sprint Goal

Establish a complete understanding of the current environment and business requirements to support a successful Microsoft Intune deployment.

---

# Sprint Objectives

- Gather business requirements.
- Assess the Microsoft 365 environment.
- Review Microsoft Entra ID.
- Review Microsoft Intune readiness.
- Assess licensing.
- Document administrative roles.
- Inventory managed devices.
- Identify implementation gaps.
