# Sprint Summary

Sprint 2 focused on understanding the organization's current environment and establishing the business and technical baseline required for the Microsoft Intune Enterprise Deployment project.

The sprint assessed the Microsoft 365 environment, Microsoft Entra ID, licensing, endpoint inventory, administrative roles, and overall readiness for Microsoft Intune implementation.

The findings from this sprint provide the foundation for the solution architecture and implementation activities that follow.

# Sprint Objectives

The objectives for Sprint 2 were:

- Document business requirements.
- Assess the current Microsoft 365 environment.
- Assess Microsoft Entra ID readiness.
- Review Microsoft licensing.
- Document the current device estate.
- Assess administrative roles.
- Identify implementation gaps.

# Key Findings

The assessment identified the following:

- Microsoft Entra ID provides a suitable identity platform for Microsoft Intune.
- Microsoft licensing supports the planned implementation.
- The organization requires centralized endpoint management.
- Device enrollment processes require standardization.
- Endpoint security can be improved through Microsoft Intune.
- Administrative responsibilities should follow Role-Based Access Control (RBAC).
- Existing management processes can be modernized through automation.

---

# Lessons Learned

The following lessons were identified during Sprint 2:

- A detailed assessment simplifies architecture design.
- Business requirements should drive technical decisions.
- Identity and licensing should always be validated before implementation.
- Administrative responsibilities should be clearly defined before deployment.
- Gap analysis provides a clear implementation roadmap.
