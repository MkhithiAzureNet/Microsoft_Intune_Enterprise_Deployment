# Project Scope

---

## Overview

This document defines the scope of the Microsoft Intune Enterprise Deployment project.

The scope establishes the project boundaries by identifying the activities, technologies, and deliverables that are included within the project, as well as those that are specifically excluded.

Defining the project scope ensures that project objectives remain aligned with business requirements while minimizing scope creep throughout the implementation lifecycle.

---

# Out of Scope

The following activities are not included within this project.

- On-premises Active Directory redesign
- Network infrastructure redesign
- Microsoft Exchange migration
- Microsoft Teams deployment
- Microsoft SharePoint implementation
- Azure Virtual Desktop deployment
- Third-party Mobile Device Management (MDM) platforms
- Non-Microsoft identity platforms
- Hardware procurement
- End-user training beyond deployment validation

---

# Project Deliverables

The project will produce the following deliverables.

- Enterprise Microsoft Intune implementation
- Device enrollment strategy
- Configuration profile library
- Compliance policy framework
- Endpoint security baseline
- Conditional Access configuration
- Application deployment framework
- Operational documentation
- Troubleshooting documentation
- Production deployment guide

---

# Scope Management

Any proposed changes to the project scope will be:

- Reviewed
- Documented
- Assessed for impact
- Approved before implementation

Scope changes will be reflected within the project roadmap and associated sprint documentation where applicable.

# Scope Approval

This document establishes the agreed project boundaries for the Microsoft Intune Enterprise Deployment project.

Any work outside this scope shall be evaluated separately before being incorporated into the project.
